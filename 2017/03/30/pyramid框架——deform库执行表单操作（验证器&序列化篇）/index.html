Pyramid框架——deform库执行表单操作（验证器&序列化篇）

3月 30, 2017 发布在 python

deform提供一些默认的带有基础功能的验证器，除默认验证器外，还可根据需要自定义验证器。deform的验证器分为两种，字段级验证器和表单级验证器。字段级验证器针对单个模式节点对象进行验证。

序列化和反序列化是针对网页显示的数据和pyramid deform表单结构的相互转化过程，序列化serialize是将appstruct结构转换成显示给用户的表单，反序列化deserialize是将用户提交的表单转换成appstruct结构给后台deform库做数据处理与验证。

默认Colander验证器

  • colander.Email 验证Email地址格式
  • colander.Length 参数长度验证
  • colander.Range 范围验证
  • colander.Any 验证子验证器是否成功验证
  • colander.Function 接收函数和可选参数的验证器（函数在验证过程中被调用）
  • colander.OneOf 验证传递值是否在集合中
  • colander.NoneOf
  • 更多默认验证器在参考文档上还有列举，如all,Regex,Containsonly,luhnok,url,uuid等

自定义验证器

除了上述默认的验证器外，我们还可以自定义验证器对表单提交的数据进行验证。字段级验证器，顾名思义，就是验证某个字段是否有问题，添加在模式节点对象的实例定义语句中；表单级验证器，综合验证多个字段间的对应关系，添加在定义schema对象实例的语句中。

执行的优先级是：先执行所有的字段级验证器，对所有字段单独验证都通过后，在执行表单级验证器，验证字段间是否正确对应。

表单提交后的验证过程

在用户提交表单之后，我们的验证器就要发挥作用了，那么如何定义提交之后的一系列动作，以及如何处理异常，编码逻辑上有几件必须要注意的点：

  • 监测提交按钮被点击的动作

Pyramid框架——deform库执行表单操作（基础流程篇）

 2017年3月30日

pyramid和deform是同一项目组开发的，所以耦合性很好，pyramid官方给的表单相关库也是deform。先上参考网址，pyramid官网上对deform库的介绍，下文的部分也是对官方文档的翻译。

deform是一个服务器端的基于python的HTML表单生成库，支持python2、python3和PyPy，支持获取日期和时间小插件、丰富的文本编辑器、动态增删条目的表单和其他一些复杂的用例。deform结合了pyramid web框架和一些其他的web框架，采用Chameleon模板和bootstrap 3来设置样式，Colander Schema 用于序列化和验证表单，Peppercorn库将HTTP获取的表单提数据变成嵌套的结构化格式。尽管deform内部使用的是Chameleon模板，但是仍然可以将渲染过的deform表单嵌入任何模板语言。
